# Nockta architecture notes — React Native

The `@react-native-community/cli init` scaffolder owns this project's
structure — `android/`, `ios/`, `App.tsx`, `index.js`, `metro.config.js`,
`babel.config.js`. The Nockta `standard` overlay for `react-native` is
deliberately minimal and disturbs nothing generated by the scaffolder.

This overlay only adds this `docs/nockta/` folder for Nockta-specific
project notes that don't belong inside the scaffolder-owned tree — native
module gotchas, platform-specific build notes, environment/config
conventions, release checklists, etc.

If you later add Expo on top of this project, the same `docs/nockta/`
overlay applies unchanged — it never touches `android/`/`ios/`/`App.tsx`.
